# mlb_prediction — Claude Code ガイド

## リポジトリ概要
MLBメジャーリーグの試合予測 HTMLを毎日生成・更新するプロジェクト。
メインファイル: `index.html`（GitHub Pages で公開）

---

## ⚠️ Git プッシュの重要ルール

### 問題
このリモート実行環境では `git push` がGitHub本体ではなく
**ローカルプロキシ（`127.0.0.1`）** に送られる。
別プロセスがGitHubを直接更新すると内容がずれる。

### 必須ワークフロー（毎回厳守）

```
1. ファイルを編集
2. git add & git commit（ローカル履歴の記録）
3. mcp__github__push_files でGitHub main ブランチへ直接プッシュ
4. mcp__github__get_file_contents で重要キーワードを検証
```

### 禁止事項
- `git push` **だけ** で完結させない（GitHub本体に届かない場合がある）
- claude/ ブランチの作成・PR の作成（mainに直接コミット）

## 日次更新フロー

### 朝版（試合開始4〜6時間前 ET 目安）
1. Web検索で予告先発・前日結果を取得
   - Baseball Reference / MLB.com / Rotowire で先発投手確認
   - 前日のスコア・勝敗を検証しrecord.mdを更新
2. 各試合の勝率を計算しindex.htmlを更新
3. ヘッダーバッジを `朝版` に設定
4. コミットメッセージ: `予測更新: YYYY-MM-DD 朝版`

### 最終分析版（試合開始1〜2時間前 ET 目安）
1. MLB.com / Baseball Reference で以下を取得
   - 先発変更の有無（injury report・lineup card）
   - 朝版で「未取得」だった投手の成績（ERA・WHIP・K/BBなど）
   - IL（故障者リスト）登録・復帰情報
2. 変動した試合は `前回比±○○pt` と理由を reason-list に明記
3. ヘッダーバッジを `最終分析版` に設定（`.final` クラスで緑色表示）
4. コミットメッセージ: `予測更新: YYYY-MM-DD 最終分析版`
5. **必ず `mcp__github__push_files` でGitHubに直接プッシュ**
6. **`mcp__github__get_file_contents` で投手名・確率が反映されているか確認**

### 試合開始済みの場合
- 未開始の試合のみ更新し、index.html内にその旨を記載

## 予測ロジック

### 基本算出手順

```
1. シーズン勝率差 → Log5法で基本勝率を算出
   Log5(A,B) = A*(1-B) / (A*(1-B) + B*(1-A))
2. 先発投手補正（ERA単独不可・複合評価）
   - 主指標: FIP（または xFIP）を最優先。ERAはBABIP・残塁率の運に左右されるため補助扱い
   - 補助指標: WHIP・K/BB・被打率(BAA)・被OPS
   - FIP差が0.5未満の場合は先発レイヤーを実質無視し確率を57%以下に抑制
   - ERA5.00以上（またはFIP4.80以上）の先発が登板する試合は55%以下に抑制
3. ブルペン恒常レイヤー（毎試合必須）
   - チームリリーフERA・FIP（シーズン通算）を左右両チームで比較
   - 前日の連投・消耗（特にクローザー・セットアッパー）と合わせて総合評価
   - ブルペン差が大きい場合は最大±3pt補正
4. 打線レイヤー
   - 主指標: wRC+（park・リーグ補正済み打撃力）または OPS+
   - 対左右スプリット（先発の投手手と相手打線のvsL/vsR成績）を照合
   - DH制による打線の厚さを加味
5. 球場パークファクター補正
   - 各球場の得点パークファクター（数値）を参照
   - ラン・スコアリング環境が平均の±10%超の球場は予想スコアに反映
6. ホームアドバンテージ +4pt（MLB長期平均）
7. 通常上限65%。チーム差・先発差・球場・対戦成績など複数の強力な根拠が重なる例外的な場合のみ最大70%まで許容（グレードA）
```

### データ取得優先順位
- **先発成績**: FanGraphs → Baseball Reference → MLB.com → ESPN の順
- **ブルペン成績**: FanGraphs（チームリリーフFIP/ERA）→ Baseball Reference
- **打線指標（wRC+）**: FanGraphs → Baseball Reference（OPS+で代替可）
- **パークファクター**: Baseball Savant → FanGraphs
- 未取得の場合は「データ未取得・保守側」として−2ptで評価。フォールバック: ERA/WHIPベースで算出

### MLB特有の注意事項
- DH制（投手打席なし）: 打線得点力の評価に用いる
- ブルペン運用: 前日の登板状況（特にクローザー・セットアッパーの連投）＋チーム全体の救援力を恒常レイヤーとして評価
- 延長戦: マナールール（10回以降・無死二塁スタート）は救援消耗を加速させる
- 同地区対戦: ライバル意識で接戦になりやすく確率を保守的に設定
- 移動・時差: 西↔東の長距離ロードトリップ・連戦疲労を考慮（特に3連戦初戦）

## index.html 構造メモ

- `.header-badge.final` — 最終分析版のときに緑色バッジ
- `.grade-b-card` / `.grade-c-card` — カード上部ラインの色
- `.upd-pos` / `.upd-neg` / `.upd-neu` — reason-list内の変動表記色
- カード順: 確率の高い順（降順）に並べる
- 順位表: AL東・AL中・AL西 / NL東・NL中・NL西 の6地区表示
- `.about-note` — サイト趣旨・賭博利用禁止の説明ブロック（免責事項の直前）

> ⚠️ **`.about-note` ブロックは変更・削除しないこと。**
> 日次更新では試合予測・確率・投手名・順位表・免責末尾の変動記録のみを更新し、このブロックは常に原文のまま維持する。

## record.md 更新ルール

- 前日試合の結果を翌日朝版更新時に記録
- 集計テーブルに日付・試合・予測side・勝率%・グレード・的中(○/×/中止)を追記
- 累積成績・グレード別的中率・Brierスコアを再計算して更新
- 雨天中止・ノーゲームは「中止」として記録し累積集計から除外

### Brierスコア算出
```
Brier = Σ(p - o)² / N
  p = 予測確率（0〜1）、o = 結果（的中=1 / 不的中=0）
  0に近いほど優秀（完全予測=0、ランダム=0.25）
```

## analysis.md 更新ルール

- 検証済み試合のうち、外れた試合と注目すべき的中試合について振り返りを記録
- 投手レイヤー・打線レイヤー・救援レイヤーのどこでズレが起きたかを分類
- 単発結果でロジックを過剰修正しない（「修正不要」と明記することも重要）
- 改善課題は「中期課題」として蓄積し、15〜20試合のサンプルが溜まってから検証
